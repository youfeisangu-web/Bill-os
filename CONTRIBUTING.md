# コントリビューションガイドライン

Bill OSへの貢献ありがとうございます！このドキュメントは、プロジェクトへの貢献方法を説明します。

## 🚀 開発の始め方

1. リポジトリをフォーク
2. ブランチを作成 (`git checkout -b feature/amazing-feature`)
3. 変更をコミット (`git commit -m 'Add some amazing feature'`)
4. ブランチにプッシュ (`git push origin feature/amazing-feature`)
5. プルリクエストを作成

## 📝 コーディング規約

### TypeScript

- `any`型の使用を避ける（可能な限り適切な型定義を使用）
- 型定義は`src/types/`に集約
- 不要な`@ts-expect-error`コメントは削除

### テスト

- 新しい機能を追加する際は、対応するテストも追加してください
- テストファイルは`src/lib/__tests__/`に配置
- テストは`npm run test`で実行可能であること

### コミットメッセージ

- 明確で簡潔なメッセージを使用
- 日本語または英語で記述
- 例: `feat: 入金消込機能に確定ボタンを追加`

## 🧪 テスト

プルリクエストを作成する前に、以下を実行してください：

```bash
# 型チェック
npm run type-check

# テスト実行
npm run test

# ビルド確認
npm run build
```

## 📋 プルリクエストのチェックリスト

- [ ] コードが既存のスタイルガイドに従っている
- [ ] テストが追加されている（該当する場合）
- [ ] すべてのテストがパスしている
- [ ] 型エラーがない（`.next/types`を除く）
- [ ] ドキュメントが更新されている（該当する場合）

## 🐛 バグレポート

バグを発見した場合、以下の情報を含めてIssueを作成してください：

- バグの説明
- 再現手順
- 期待される動作
- 実際の動作
- 環境情報（OS、Node.jsバージョンなど）

## 💡 機能リクエスト

新機能の提案も歓迎します！Issueで以下の情報を提供してください：

- 機能の説明
- なぜこの機能が必要か
- 実装の提案（あれば）

## 📄 ライセンス

コントリビューションにより、あなたのコードはプロジェクトのライセンスの下で公開されることに同意したものとみなされます。
